# 任务A完整准备与可运行基线方案

## Summary
在项目根目录搭建一套可直接用于任务A的完整基线工程，覆盖 `GSM8K + CSQA + MMLU` 三个数据集，以及 `Qwen2.5-7B-Instruct / Mistral-7B-Instruct-v0.3 / Olmo-3-7B-Instruct` 三个模型。交付内容包含：
- `docs/A` 下的任务A执行清单
- `fjm_CoT` conda 环境定义与安装脚本
- 模型下载脚本、数据集下载脚本、统一目录结构
- Zero-shot CoT、Few-shot CoT、Self-Consistency 三类推理脚本
- 结果日志、配置文件、提示词模板、冒烟测试脚本
- 一套“可跑通但不启动正式实验”的 smoke test 流程

准备完成后的目标状态：
- 在新环境里一条命令完成依赖安装
- 三个数据集和三个模型都能下载到项目根目录指定位置
- 所有任务A脚本都能被调用并完成最小样本推理
- 冒烟测试验证每个模块可运行，先不跑全量实验

## Key Changes
### 1. 目录与文档
在项目根目录建立固定结构，避免后续 B/C 接手时再改接口：
- `docs/A/`
  - `task_A_checklist.md`：任务A清单，写清目标、步骤、输入输出、完成标准、命令示例
- `configs/`
  - `models/`：三个模型的配置文件
  - `datasets/`：三个数据集的配置文件
  - `prompts/`：zero-shot、few-shot、self-consistency 相关模板和参数
  - `runs/`：冒烟测试与正式实验的运行配置
- `scripts/`
  - 环境安装、模型下载、数据下载、数据预处理、运行推理、运行冒烟测试
- `src/`
  - 数据集加载模块
  - Prompt 构造模块
  - 模型加载与推理模块
  - Self-Consistency 投票模块
  - 结果保存模块
- `data/`
  - `raw/`、`processed/`
- `models/`
  - 三个模型各自的本地目录
- `outputs/`
  - `logs/`、`predictions/`、`smoke_tests/`

### 2. 环境与依赖
创建 `fjm_CoT` conda 环境，环境方案固定为：
- Python 3.10
- PyTorch GPU 版本，适配 CUDA 12.x 的官方 wheel
- `transformers`、`accelerate`、`datasets`、`sentencepiece`、`protobuf`、`scipy`、`pandas`、`tqdm`、`pyyaml`、`evaluate`
- `bitsandbytes` 作为 4bit/8bit 推理依赖，保证 7B/8B 模型在显存受限时也能跑最小推理
- `huggingface_hub` 用于模型和数据集下载
- `pytest` 用于冒烟测试入口

交付形式：
- `environment.yml`
- `requirements.txt`
- `scripts/setup_env.sh`
- `scripts/verify_env.sh`

环境验证标准：
- 可以成功 import 全部核心依赖
- 可以识别 GPU
- 可以加载至少一个量化模型并完成单题推理

### 3. 模型准备
固定接入以下 3 个模型：
- `Qwen/Qwen2.5-7B-Instruct`
- `mistralai/Mistral-7B-Instruct-v0.3`
- `allenai/Olmo-3-7B-Instruct`

模型下载策略：
- 使用 `huggingface_hub.snapshot_download`
- 下载到项目根目录 `models/` 下的独立子目录
- 每个模型有独立配置文件，记录本地路径、dtype、量化方式、最大生成长度、是否启用采样

模型运行接口统一：
- 输入：题目、prompt 模板、采样参数
- 输出：完整推理文本、最终答案、元信息
- 统一支持三种模式：
  - `zero_cot`
  - `few_shot_cot`
  - `self_consistency`

Self-Consistency 固定行为：
- 单题采样 `N` 条路径
- 每条路径抽取最终答案
- 用未加权多数投票得到最终预测
- 保存全部路径与投票明细
- 不在任务A阶段加入复杂度筛选、归一化投票、语义投票

### 4. 数据集准备
固定接入：
- `GSM8K`
- `CommonsenseQA`
- `MMLU`

数据准备内容：
- 下载原始数据到 `data/raw/`
- 转换为统一 JSONL 格式到 `data/processed/`
- 统一字段：
  - `id`
  - `dataset`
  - `question`
  - `choices`
  - `answer`
  - `metadata`
- 对 MMLU 固定为标准任务格式，保留学科字段
- Few-shot 示例从训练集或开发集按固定规则抽取，并保存为本地 prompt 资源文件，避免每次运行动态变化

任务A阶段的数据处理边界：
- 只做推理所需的标准化
- 不做额外清洗、增强或重采样
- 对答案抽取规则按数据集分别实现，直接暴露异常，不加兜底静默修复

### 5. 脚本与运行流
需要写齐的脚本：
- `scripts/setup_env.sh`
  - 创建并安装 `fjm_CoT`
- `scripts/download_models.py`
  - 下载三模型到本地
- `scripts/download_datasets.py`
  - 下载三数据集并标准化
- `scripts/build_fewshot_prompts.py`
  - 生成固定 few-shot 示例文件
- `scripts/run_zero_shot.py`
- `scripts/run_few_shot.py`
- `scripts/run_self_consistency.py`
- `scripts/run_smoke_tests.sh`

代码层统一接口：
- `src/data/`：按数据集读取统一样本对象
- `src/prompts/`：根据模式和数据集生成 prompt
- `src/models/`：封装本地模型加载和 generation
- `src/sc/`：多路径采样、答案抽取、多数投票
- `src/io/`：输出 JSONL / CSV / 日志

输出文件格式统一：
- 每条样本保存：
  - 输入题目
  - prompt
  - 原始生成文本
  - 抽取答案
  - gold answer
  - 是否正确
- SC 额外保存：
  - 所有路径
  - 每条路径答案
  - 投票计数
  - 最终投票答案

### 6. 冒烟测试
只做“能运行”验证，不跑正式实验。

冒烟测试覆盖：
- 环境安装检查
- 每个数据集都能成功读取 1 到 3 条样本
- 每个模型都能成功加载
- Zero-shot CoT：每个模型在 `GSM8K` 上跑 1 题
- Few-shot CoT：每个模型在 `CSQA` 上跑 1 题
- Self-Consistency：每个模型在 `MMLU` 上跑 1 题，`N=3`
- 所有结果成功写入 `outputs/smoke_tests/`

通过标准：
- 所有脚本返回码为 0
- 每次运行都能产出结构正确的结果文件
- Self-Consistency 能产出多条路径和投票结果
- 不要求准确率达标，不启动全量任务

## Public Interfaces / Types
本次要固定下来的外部接口：
- 环境入口：`conda activate fjm_CoT`
- 统一运行命令：
  - `python scripts/run_zero_shot.py --config ...`
  - `python scripts/run_few_shot.py --config ...`
  - `python scripts/run_self_consistency.py --config ...`
- 统一结果格式：JSONL 为主，CSV 汇总为辅
- 统一配置格式：YAML
- 统一样本结构字段：
  - `id`
  - `dataset`
  - `question`
  - `choices`
  - `gold_answer`
  - `prediction`
  - `reasoning`
  - `correct`

- 数据测试：三个数据集下载成功并能转为统一格式

## Assumptions
- 模型范围固定为 `Qwen2.5-7B-Instruct + Mistral-7B-Instruct-v0.3 + Olmo-3-7B-Instruct`
- 数据范围固定为 `GSM8K + CSQA + MMLU`
- 模型与数据都实际下载到项目根目录下的 `models/` 和 `data/`
- 任务A只做 baseline，不引入复杂度筛选、答案归一化、语义投票等改进
- 允许使用量化推理来保证 7B/8B 模型能稳定做冒烟测试
- 冒烟测试只验证“脚本、环境、下载、推理链路”可运行，不跑正式实验
- 代码需带详细中文注释，模块边界清晰，直接暴露真实异常，不额外添加未要求的兜底逻辑
